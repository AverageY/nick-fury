client root
1. get /client : List operation, gives all client
         response schema(List<Client>)
        200: [{“ClientId”:”1”,“Name”:”Yash”,Email:”yash@gmail.com”,”CreatedAt”:”iso 8601 time”},...}]
        -Includes response body schema for 4xx and 5xx error codes
        
2. post /client: creates a new client
request schema(ClientDto) : {“name”:”Yash”,Email:”yash@gmail.com”,”Password”:”pass”}
response schema:
201: includes a custom schema(ApiResponse) for account creation with {status:completed, message:account created}
- includes response for 4xx and 5xx (uses ApiResponse)


3. get /client/{clientId}: fetches client by id
params: clientId
response schema(Client):{“ClientId”:”1”,“name”:”Yash”,Email:”yash@gmail.com”,”CreatedAt”:”iso 8601 time”}
-response for client doesnt exist
-includes response for 4xx and 5xx (uses ApiResponse)


4. put /client/{clientId}: updates a client
params: clientId
request schema(ClientUpdateDto):
{“name”:”Yash”,Email:”yash@gmail.com”,”OldPassword”:”pass”,”NewPassword”:”NewPass”} —none of the fields are set as required here since we dont know what user will update
response schema: uses the ApiResponse schema with status and message:client updated
-response for client doesnt exist
-includes response for 4xx and 5xx (uses ApiResponse)


5. delete /client/{clientId}:
params:ClientId
response : Deleted successfully
-response for client doesnt exist and 4xx 5xx errors

root message:
1. post at /message/{clientId}/{therapistId}:
params: clientId , therapistId
request schema(MessageDto) :
{
“Content”:”message content here”,
“Sender”:”sender name”}
comments - The message Id will be clientId#therapistId so we can retrieve all the messages of a conversation between a client and therapist
sender name - we can retrieve that through the sender id, checking the id in client/therapist table
We plan to use the partition key as MessageId and Timestamp as range key to get message in sorted order
response: Api Schema for 201
-Error responses for 3xx,4xx,5xx(client,therapist etc not found)


2. get at  /message/{clientId}/{therapistId}:
params: clientId , therapistId
request schema(MessageViewDto) :
{“Timestamp”:”date in iso8601”,
“Content”:”message content here”,
“Sender”:”sender name”}
